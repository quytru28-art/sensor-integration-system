SIS Word Documents — Alignment With Test_Plan_template / Test Script_template
and With Test_Plan_example / Test Script_example (HMMS)
================================================================================

SUMMARY
-------
• SIS Test Plan follows the same MAJOR OUTLINE as the Test Plan EXAMPLE (HMMS):
  sections 1–3 (intro, strategy, environment) then 4 Test Identification,
  5 Test Schedules, 6 Requirements Traceability, 7 Risks. That matches the
  example’s body structure, not the template’s table of contents (which moves
  test identification into Appendix C).

• SIS Test Plan differs from the generic TEMPLATE in numbering and in missing
  a few template subsections (Resources, Referenced documents, Readiness
  criteria, formal appendices A–C).

• SIS Test Scripts follow the same LOGICAL TABLE as the Test Script template
  and example (test case → procedure → expected result → pass/fail), but SIS
  adds fields the minimal template omits (Script ID, Severity, Preconditions)
  and does not repeat the template’s cover fields (Database, Tester Name,
  Revision, Dates, Execution times) at the top of the file.

IF YOUR RUBRIC REQUIRES STRICT TEMPLATE MATCHING
------------------------------------------------
Add in Word (copy/paste style from Test_Plan_template):

Test Plan — after Document Overview:
  • "1.4 Resources" — who tests, roles (Tester, Dev, PM).
  • "1.5 Referenced documents" — link User Stories, SRS, design repo, SIS_Test_Scripts.

Test Plan — inside Strategy:
  • "2.5.1 Readiness Criteria" (or renumber): test plan approved, environment up,
    test accounts ready, scripts reviewed.

Test Plan — end matter:
  • "Appendix A – Acronym List" (SIS, JWT, API, CRUD, SMTP, …).
  • "Appendix B – Test report" — blank form or pointer to Pass/Fail log.

Test Scripts — before TS-01:
  • One cover table from Test Script_template (Database: SQLite sensor_system.db;
    Tester Name; dates; Revision 1.0).

CONCLUSION
----------
SIS documents are ALIGNED IN SUBSTANCE with the HMMS-style EXAMPLE (especially
the test plan sections 4–7). They are NOT a line-for-line match to the generic
TEMPLATE TOC; the gaps above are normal for a project-specific plan and can be
filled with short insertions if the course requires literal template compliance.
